DNS Problem

If your domain can’t resolve to your github pages, a possible solution is as following [1] [5] . First run the following command (replace deepperand to your github user name!)

then add the output ip to your DNS record with type A (left hostname empty). More knowledge about record type of DNS resolve can see here [2] .

Can’t Enable HTTPS

For me, this problem is tightly associate with DNS problem. However, there need some other operations, but the I didn’t knowledge the principle of them. The most helpful operation includes [5] [6]

  • delete custom domain (for me is www.deeperand.xyz) at Settings -> Pages -> Custom domain and save
  • fill the domain again
  • try to access website with https (for me is https://www.deeperand.xyz)

if the option is still can’t enable, check if the https certification is already generated, since a new certification for GitHub pages need some time (less than 24H) to generate [5] . If it also helpful, try to repeat the above operation, then it might can enable https dramatically.
