[DEBUG] `vim/macvim` Crash when use Python

最后更新：2022.03.10, 17:59

上周二晚上因为觉得没配置过的 macvim 太难用, 从旧电脑上把 vimrc 的代码复制到了新电脑, 让新电脑的 vim 和 macvim 在加载时也采用 blade-vim [1] .

但谁料这之后一旦涉及到使用 pythin3 的场景就发生闪退. 例如在 vim 中使用如下代码以检查 Python 版本时 [2]

经查询后发现, 问题可能是进程内的环境变量 $PYTHONHOME 的设置问题. 但令人费解的是, 我的 bash 环境里中并没有设置过这个变量. 经排查发现, 问题在于 vimrc 的一段旧代码上 (之所以确定是旧代码, 是因为其在以前的机器上也出现了) 设置了一些与 python 相关的变量, 将其注释掉就解决问题了.
