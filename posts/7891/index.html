Bacis Knowledge of Tmux

最后更新：2021.11.09, 10:19

the main content is from my notes about tmux written from 2019 November to 2021 January

Basic Knoledge

  1. 最系统的配置说明是 Tmux 自己的 manual 页, 可通过 man tmux 来查看.
  2. 主要的结构层次为: server >? session > window > pane
  3. 同一行执行多条语句使用 ;, 将两行拼接成一行使用 \
  4. Command arguments may be specified as strings surrounded by single (‘) quotes, double quotes (“) or braces ({}). the main difference is:
    1. 单引号及花括号中的特殊字符保持字面义. 特别地, 花括号行末的续接符 \ 也将保持字面义
    2. 双引号中诸如 $ 等符号具有特殊的意义, 如调用环境变量
    3. 花括号允许直接换行, 而单引号及双引号若要换行, 需要在行末加上反斜杠 \
  5. the rule of escape characters:
    • Variable : Environment variables preceded by $ are replaced with their value from the global environment (see the GLOBAL AND SESSION ENVIRONMENT section).
    • Home Direction : A leading ~ or ~user is expanded to the home directory of the current or specified user.
    • Unicode Character :\uXXXX or \uXXXXXXXX is replaced by the Unicode codepoint corresponding to the given four or eight digit hexadecimal number.
    • Escaped Character : When preceded (escaped) by a \, the following characters are replaced: \e by the escape character; \r by a carriage return; \n by a newline; and \t by a tab.
    • Octal Character : \ooo is replaced by a character of the octal value ooo. Three octal digits are required, for example \001. The largest valid character is \377.
    • Other Rules : Any other characters preceded by \ are replaced by themselves (that is, the \ is removed) and are not treated as having any special meaning —- so for example \; will not mark a command sequence and \$ will not expand an environment variable.
  6. 文件自动加载机制 : 每次启动 Tmux 时, 其会自动加载文件 ~/.tmux.conf

Variable

  1. Tmux 允许变量赋值的语句, 支持使用 $ 对变量进行调用. 应注意, 这里的变量不是 Shell 的变量, 而是 Tmux 自己的变量
  2. 以 # 开头的一般为特殊变量, 调用这些变量时不需要额外加上 $ . 重要的有:
    1. 一些内置的变量及其别名, 如 #{host}, 其别名为 #H

启用外部程序

  1. #(<shell_command>) 来使该处变为 <shell_command> 的输出结果, 但只有在第二次调用相同的命令时, 才会将该处的字符串替换为上一次的输出结果. 若想在脚本中使用, 或许一个好的主义是将原始的文件 source 两次? 这样的话就可以保证第二次一定能正确输出结果.
  2. #(<shell_command>) 只取最后一次的输出结果. 例如: #(echo 1; echo 2) 只会将该处替换为 2.
  3. 似乎括号里面相当与一个子进程? 测试表明, 如果在文件 foo.sh 中定义了函数 foo, 则使用 #(source foo.sh; foo) 可以得到 foo 的输出结果

Notation :

  1. 总体来说, Tmux 的环境分为两种: global enviroment, session enviroment
  2. Tmux 在启动时, 会从 Shell 的环境变量中继承变量.

Command

为简化书写, 以下的指令都省略了 tmux. 既是说, 标注为 new-window 的指令在 shell 中执行时, 应输入 tmux new-window

  1. new-window [-adkP] [-n window-name] [-t target-window] [shell-command]: (alias: neww) Create a new window.
    1. -a: With \a, the new window is inserted at the next index up from the specified target-window, moving windows up if necessary, otherwise target-window is the new window location.
    2. -d, \k: If -d is given, the session does not make the new window the current window. target-window represents the window to be created; if the target already exists an error is shown, unless the -k flag is used, in which case it is destroyed. shell-command is the command to execute. If shell-command is not specified, the value of the default-command option is used.
    3. -P: The -P option prints the location of the new window after it has been created.
    4. When the shell command completes, the window closes. See the remain-on-exit option to change this behaviour.
    5. The \$TERM environment variable must be set to screen for all programs running inside tmux. New windows will automatically have TERM=screen added to their environment, but care must be taken not to reset this in shell start-up files.
  2. display <message>: Print the message at the status line, just like: tmux display "a message", then the text a message will be displayed at the status line of Tmux.
  3. list-windows [-a] [-Fformat] [-ttarget-session] (alias lsw): If -a is given, list all windows on the server. Otherwise, list windows in the current session or in target-session. For the meaning of the -F flag, see the FORMATS section.
  4. capture-pane [-aepPqCJ] [-b buffer-name] [-E end-line] [-S start-line] [-t target-pane]: Capture the contents of a pane.
    1. -p, b: If -p is given, the output goes to stdout, otherwise to the buffer specified with -b or a new buffer if omitted.
    2. -a, q: If -a is given, the alternate screen is used, and the history is not accessible. If no alternate screen exists, an error will be returned unless -q is given.
    3. -e: If -e is given, the output includes escape sequences for text and background attributes.
    4. -C: -C also escapes non-printable characters as octal \xxx.
    5. -P: -J joins wrapped lines and preserves trailing spa- ces at each line’s end.
    6. -J: -P captures only any output that the pane has received that is the beginning of an as-yet incomplete escape sequence.
    7. -S, -E: -S and -E specify the starting and ending line numbers, zero is the first line of the visible pane and negative numbers are lines in the history. `-‘ to -S is the start of the history and to -E the end of the visible pane. The default is to capture only the visible contents of the pane.
  5. if-shell [-bF] [-t target-pane] shell-command command [command]: (aliais: if) Execute the first command if shell-command returns success or the second command otherwise. Before being executed, shell-command is expanded using the rules specified in the FORMATS section, including those relevant to target-pane.
    1. -b: With -b, shell-command is run in the background.
    2. -F: If -F is given, shell-command is not executed but considered success if neither empty nor zero (after formats are expanded).
  6. run-shell [-b] [-t target-pane] shell-command: Execute shell-command in the background without creating a window. Before being executed, shell-command is expanded using the rules specified in the FORMATS section.
    1. -b: With -b, the command is run in the background. After it finishes, any output to stdout is displayed in copy mode (in the pane specified by -t or the current pane if omitted). If the command doesn’t return success, the exit status is also displayed.

Option Related

  1. set-option: 使用指令 set-option 或 set 设置选项, 其后可加参数:
    1. -g: 意为 “global”, 表示该设置作用于所有 session
    2. -s: 意为 “server”
    3. -w: 意为 “window”
    4. -u: unset a option
    5. -a: if the option expects a string, value is appended to the existing setting
  2. show-options [-AgHpqsvw] [-t target-pane] [option]: (alias: show)
    1. -p, -w, -s: Show the pane options (or a single option if [option] is provided) with -p, the window options with -w, the server options with -s, otherwise the session options. If the option is not a user option, -w or -s may be unnecessary - tmux will infer the type from the option name, assuming -w for pane options.
    2. -g: Global session or window options are listed if -g is used.
    3. -v: -v shows only the option value, not the name.
    4. -q: If -q is set, no error will be returned if option is unset.
    5. -H: -H includes hooks (omitted by default).
    6. -A: -A includes options inherited from a parent set of options, such options are marked with an asterisk. value depends on the option and may be a number, a string, or a flag (on, off, or omitted to toggle).

Notation :

  1. setw 是 set-window-option 的简称. 现暂时不了解其是否等效于 set-option -w

Variable Environment Related

  1. set-environment [-gru] [-t target-session] name [value]: (alias: setenv) Set or unset an environment variable.

    1. -g, -t: If -g is used, the change is made in the global environment; otherwise, it is applied to the session environment for target-session.
    2. -u: The -u flag unsets a variable.
    3. -r: indicates the variable is to be removed from the environment before starting a new process.

  2. show-environment [-gs] [-t target-session] [variable]: (alias: showenv) Display the environment for target-session

    1. -g: display the global environment with -g.
    2. -s: If -s is used, the output is formatted as a set of Bourne shell commands.
    3. If variable is omitted, all variables are shown.
    4. Variables removed from the environment are prefixed with -.

Format

Description : Certain commands accept the -F flag with a format argument. This is a string which controls the output format of the command. Format vari‐ ables are enclosed in #{ and }, for example #{session_name}. The possible variables are listed in the table below, or the name of a tmux option may be used for an option’s value. Some variables have a shorter alias such as #S; ## is replaced by a single #, #, by a , and #} by a }.

也就是说如果在命令中看到形如 #{} 的表达式, 那一般来说是和格式控制有关的. 这并非是 shell 的语法, 而是 Tmux 配置文件的语法. 此外, 形如 #D, 亦即 # 符号后跟一个字母的, 一般来说是 format variable 的别名. 详细的 format variable 的信息见 man tmux.

Option Setting

The Type of Options

The appearance and behaviour of tmux may be modified by changing the value of various options. There are four types of option:

  1. server options
  2. session options
  3. window options
  4. pane options
    The tmux server has a set of global options which do not apply to any particular window or session or pane. These are altered with the set-option -s command, or displayed with the show-options -s command.

In addition, each individual session may have a set of session options, and there is a separate set of global session options. Sessions which do not have a particular option configured inherit the value from the global session options. Session options are set or unset with the set-option command and may be listed with the show-options command. The available server and session options are listed under the set-option command.

Similarly, a set of window options is attached to each window and a set of pane options to each pane. Pane options inherit from window options. This means any pane option may be set as a window option to apply the option to all panes in the window without the option set.

There is also a set of global window options from which any unset window or pane options are inherited. Window and pane options are altered with set-option -w and -p commands and displayed with show-option -w and -p.
